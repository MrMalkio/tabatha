Data & Analytics

Every chart on this page is hand-rolled: the extension ships no chart library, so each bar, cell and track is a plain div with a CSS width or height. That constraint is the whole aesthetic here, from the 365-day activity heatmap down to the 24 hour-of-day slivers in Peak Productivity Hours.

Analytics surfaces

15 components · dashboard, heatmap, work shifts, logs

What the tracking is for. If Tabatha only ever showed you a running clock it would be a stopwatch; these components turn stints into something you can act on - which hours are actually productive, where the day fragments, what a context really costs. The reporting bias is toward patterns over totals, because a weekly total tells you nothing you can change. Worth noting: every chart here is hand-rolled CSS. The extension ships no charting library at all, which keeps the bundle small and the rendering identical across all twelve themes.

ActivityHeatmap
365 days as 11px cells, seven rows deep, switchable between browser time, browser plus desktop, and follow-through actions. One real quirk is reproduced faithfully: levels 1 to 3 read rgba(var(--accent-rgb, 59,130,246), …) but --accent-rgb is never defined anywhere in the codebase, so the fallback always wins and the mid-levels render blue in every theme.
